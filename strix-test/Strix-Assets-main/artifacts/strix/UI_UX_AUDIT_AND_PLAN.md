# تقرير مراجعة واجهة Strix (UI/UX) وخطة التطوير

> **الغرض:** مراجعة تصميم التطبيق وتجربة المستخدم، مع خطة تنفيذ مفصّلة لنموذج/مطوّر أقل خبرة.
>
> **قيد أساسي من العميل:** **نفس الألوان الحالية** — لا نغيّر الهوية اللونية (أخضر Strix `#1DB768`، الأحمر، البرتقالي…). كل ما نفعله هو **توحيد** استخدام هذه الألوان وإصلاح التضارب، لا تغييرها.
>
> **النطاق:** `app/**` (الشاشات)، `components/**`، `constants/colors.ts`، `lib/rtl.ts`، `context/LanguageContext.tsx`، `lib/i18n` + `lib/locales/*`.
>
> **تاريخ المراجعة:** 2026-06-26

---

## 0. قواعد للنموذج المنفّذ (اقرأها أولاً)

1. **لا تغيّر قيم الألوان.** فقط استبدل الأكواد اللونية المكتوبة يدوياً (hardcoded hex) بالـ tokens المعرّفة في `constants/colors.ts`. النتيجة البصرية يجب أن تبقى نفسها أو أقرب ما يكون.
2. **لا تكسر RTL.** المشروع يدير RTL يدوياً عبر `lib/rtl.ts` و`I18nManager.forceRTL(false)`. لا تستدعِ `forceRTL(true)` أبداً.
3. **كل نص يراه المستخدم يجب أن يمرّ عبر `t()`** — ممنوع نص عربي/إنجليزي ثابت في الكود أو الإشعارات.
4. **بعد كل تعديل:** `npx tsc --noEmit` يجب أن يبقى نظيفاً، و`npx jest` أخضر.
5. نفّذ حسب الأولوية في الجدول (القسم 2). لا تخلط عدة إصلاحات في commit واحد.
6. لا تضف مكتبات ثقيلة جديدة دون داعٍ.

## 1. ملخّص الحالة الحالية

**نقاط القوة (نبني عليها):**
- نظام RTL يدوي متّسق عبر `rtl.flexDirection` / `rtl.textAlign` + دالة `flipIconName` لقلب الأسهم.
- خطوط ثنائية اللغة صحيحة: Inter للإنجليزي و IBM Plex Sans Arabic للعربي، مُدارة مركزياً في `components/Text.tsx`.
- طبقة ترجمة منظّمة (`i18next`) + دوال تنسيق أرقام محلية في `LanguageContext` (`formatNumber`, `formatGForce`, `formatPercentage`…).
- حالات فارغة (Empty States) موجودة في الرئيسية.
- هوية لونية واضحة (أخضر Strix) ونظام `radius` موحّد.

**أهم المشاكل (سنصلحها):**
1. النمط الداكن مُعطّل فعلياً، مع ألوان داكنة مكتوبة يدوياً فوق خلفية فاتحة → تضارب بصري.
2. نفس اللون الدلالي مكتوب بعدة درجات مختلفة عبر الملفات (أخضر `#1DB768` مقابل `#3FB950` مقابل `#34C759`؛ أحمر `#FF3B30` مقابل `#FF4444`).
3. نصوص عربية ثابتة تظهر حتى عند اختيار الإنجليزية (إشعارات، تحليل الطرف الآخر، ملاحظات).
4. رمز النسبة `٪` ثابت، وأرقام لا تمرّ على التنسيق المحلي.
5. تباين ألوان النص الثانوي (`mutedForeground #8A9A8A`) على الأبيض يفشل في معايير الوصول (WCAG AA).
6. أزرار بلا `accessibilityLabel` (قارئ الشاشة لا يفهمها).

## 3. تفاصيل المشاكل وخطوات الإصلاح

### 🔴 U-1 — توحيد الألوان (نفس الألوان، مصدر واحد)

**التشخيص:** نفس اللون الدلالي مكتوب بدرجات مختلفة عبر الملفات:
- أخضر: `#1DB768` (palette) ، `#3FB950` (report/session) ، `#34C759` (ReportCard/report).
- أحمر: `#FF3B30` (palette) ، `#FF4444` (session/report).
- برتقالي: `#FF9500` (palette warning) ، `#FF9340` (ReportCard/report).
- رمادي داكن: `#8B949E` ، `#30363D` (من ثيم داكن قديم) فوق خلفية فاتحة.

هذا يجعل التطبيق يبدو غير مصقول، ويصعّب أي تعديل مستقبلي.

**الإصلاح:**
1. أضف ألوان الشدّة (severity) كـ tokens في `constants/colors.ts` ضمن `light` و`dark` (بنفس القيم الحالية لتفادي تغيير المظهر):
```typescript
// داخل light (وكرّرها في dark بنفس القيم أو الأنسب للداكن)
severityCritical: "#FF3B30",
severitySevere:   "#FF3B30",
severityModerate: "#FF9500",
severityMinor:    "#1DB768",
```
2. أنشئ خريطة موحّدة `lib/severityColors.ts`:
```typescript
import type { Severity } from "./types";
export function severityColor(c: { severityCritical: string; severitySevere: string; severityModerate: string; severityMinor: string }, sev: Severity): string {
  return { critical: c.severityCritical, severe: c.severitySevere, moderate: c.severityModerate, minor: c.severityMinor }[sev];
}
```
3. استبدل كل `SEVERITY_COLOR` المحلية في `session.tsx` و`report/[id].tsx` و`ReportCard.tsx` بالاستدعاء الموحّد.
4. استبدل كل `#3FB950` و`#34C759` بـ `colors.success`، و`#FF4444` بـ `colors.destructive`، و`#FF9340` بـ `colors.warning`، و`#8B949E` بـ `colors.mutedForeground`.
5. ابحث عن كل hex يدوي وتأكد أن له token مقابل:
```bash
grep -rnE "#[0-9A-Fa-f]{6}" app components | grep -v constants/colors.ts
```

**معيار النجاح:** لا يوجد hex لوني خارج `constants/colors.ts` (باستثناء الشفافيات مثل `colors.primary + "18"`). المظهر لم يتغيّر بصرياً.

### 🔴 U-2 — حسم النمط الداكن

**التشخيص:** `hooks/useColors.ts` يُرجع **دائماً** `colors.light` ويتجاهل `useColorScheme()`، بينما `colors.dark` معرّف لكنه غير مستخدم. وفي نفس الوقت توجد ألوان داكنة ثابتة (`#120B0B` لبطاقة الحادث، `#0D1117` لشاشة البداية، `#8B949E`, `#30363D`). النتيجة: التطبيق فاتح لكن فيه عناصر داكنة دخيلة.

**القرار المطلوب (اختر A أو B):**
- **A — تفعيل النمط الداكن فعلياً (موصى به):**
```typescript
// hooks/useColors.ts
import { useColorScheme } from "react-native";
import colors from "@/constants/colors";
export function useColors() {
  const scheme = useColorScheme();
  const palette = scheme === "dark" ? colors.dark : colors.light;
  return { ...palette, radius: colors.radius };
}
```
  ثم أزل كل لون داكن ثابت (مثل `#120B0B` في `crashCard`) واستبدله بـ `colors.card`. هكذا يعمل النمطان بنفس الهوية.
- **B — قفل النمط الفاتح فقط:** أبقِ `useColors` كما هو، لكن **أزل كل الألوان الداكنة الثابتة** واستبدلها بـ tokens فاتحة (مثلاً `crashCard` خلفيتها `colors.card` بدل `#120B0B`، والنص `colors.foreground`).

> أياً كان الاختيار، **يجب ألّا تبقى ألوان داكنة ثابتة فوق خلفية فاتحة.** هذا أصل التضارب.

**أمثلة على ما يجب استبداله (في `session.tsx`):**
- `crashCard.backgroundColor: "#120B0B"` → `colors.card`.
- نصوص `#F0F6FF` → `colors.foreground`؛ `#8B949E` → `colors.mutedForeground`؛ `#30363D` → `colors.border`.
- في `_layout.tsx` شاشة الانتظار `backgroundColor: "#0D1117"` → `colors.background` (استورد الـ palette أو استخدم أبيض ثابت متوافق).

**معيار النجاح:** لا ألوان داكنة ثابتة؛ تبديل نمط الجهاز (إن فُعّل A) يبدّل الواجهة كاملةً دون بقع.

### 🔴 U-3 — نصوص عربية ثابتة تظهر في الإنجليزي

**التشخيص:** أماكن تكتب عربيّاً مباشرةً بغضّ النظر عن اللغة:
1. `context/SessionContext.tsx` — إشعار الخلفية:
   ```
   notificationTitle: "ستركس يراقب القيادة",
   notificationBody: "مراقبة الحوادث نشطة في الخلفية",
   ```
2. `context/SessionContext.tsx` — ملاحظة تعديل النسبة:
   ```
   updatedDesc += "\n(تم تعديل النسبة بعد مقاطعة البيانات ...)";
   ```
3. `lib/otherPartyAnalysis.ts` — كامل `descriptionAr` وكلمات `vehicleAr`/`forceAr`/الاتجاهات مكتوبة عربيّاً.
4. `lib/liabilityEngine.ts` — ملاحظة الاتجاه غير المعاير التي أُضيفت في A-6 (نص عربي حرفي).

**الإصلاح:**
1. أضف مفاتيح ترجمة جديدة في `lib/locales/ar.json` و`en.json`، مثال:
```json
// notifications
"notifications": {
  "monitoringTitle": "...", "monitoringBody": "..."
},
"report": { "crossAdjustedNote": "..." },
"liability": { "directionUncalibratedNote": "..." }
```
2. الإشعار: مرّر النصوص عبر `i18n.t(...)`. لكن طبقة `lib/` لا تملك سياق React — استخدم `i18n.t` مباشرةً من `import i18n from "@/lib/i18n"` (نفس ما يفعله `liabilityEngine`).
3. `otherPartyAnalysis`: حوّل `buildDescription` لاستخدام `i18n.t` مع متغيّرات (interpolation)، بدل تركيب السلسلة عربيّاً. وانقل `vehicleAr/forceAr/dirAr` إلى مفاتيح ترجمة.
4. أعد تسمية الحقول التي تحمل لاحقة `Ar` إلى محايدة (مثلاً `description`) تدريجياً، أو أبقِ الاسم لكن املأه باللغة الحالية.

> **ملاحظة مهمة:** بعض هذه النصوص تُحفظ داخل التقرير وقت الحادث (snapshot). القرار: هل التقرير ثابت اللغة وقت إنشائه، أم يُترجم وقت العرض؟ الأفضل **الترجمة وقت العرض** (احفظ مفاتيح/مدخلات لا جُملاً جاهزة) — لكن هذا تغيير أوسع؛ كبداية اجعلها تتبع اللغة الحالية وقت الإنشاء.

**معيار النجاح:** بدّل اللغة إلى الإنجليزية وتصفّح كل الشاشات + الإشعار → لا تظهر أي كلمة عربية.

### 🟠 U-4 — رمز النسبة `٪` الثابت والأرقام غير المحلية

**التشخيص:** في `report/[id].tsx` و`ReportCard.tsx`:
```tsx
<Text>{report.liabilityScore}٪</Text>   // ٪ ثابت ويُظهر أرقام لاتينية بلا تنسيق
{displayReport.speedKmh} {t("report.kmh")}  // رقم خام
```

**الإصلاح:** استخدم دوال `LanguageContext`:
```tsx
const { formatPercentage, formatNumber } = useLanguage();
// ...
<Text>{formatPercentage(report.liabilityScore)}</Text>
<Text>{formatNumber(displayReport.speedKmh)} {t("report.kmh")}</Text>
```
`formatPercentage` يختار `٪` للعربي و`%` للإنجليزي والأرقام الصحيحة لكل لغة.

**معيار النجاح:** بالإنجليزية تظهر `75%` بأرقام لاتينية، وبالعربية `٧٥٪` (حسب نظام الأرقام المختار).

### 🟠 U-5 — تباين النص الثانوي (وصول)

**التشخيص:** `mutedForeground: "#8A9A8A"` على خلفية بيضاء `#FFFFFF` نسبة تباينه ≈ 2.6:1، وهو **أقل من حدّ WCAG AA** (4.5:1 للنص الصغير). يُستخدم في التواريخ، التسميات، الوصف — أي نصوص كثيرة يصعب قراءتها على بعض المستخدمين/الإضاءة.

**الإصلاح (مع الحفاظ على الإحساس الرمادي):** غمّق درجة الرمادي قليلاً حتى يجتاز AA:
```typescript
// constants/colors.ts (light)
mutedForeground: "#5F6B5F",  // بدل #8A9A8A — يجتاز AA على الأبيض وعلى الكرت
```
وفي `dark` أبقِ `#8A9A8A` (يجتاز على الخلفيات الداكنة).

> هذا تعديل درجة واحدة ضمن نفس العائلة الرمادية، لا يغيّر الهوية. تحقّق بأي فاحص تباين (هدف ≥ 4.5:1).

**معيار النجاح:** كل النصوص الثانوية تجتاز 4.5:1 على خلفيتها.

### 🟠 U-6 — تسميات الوصول على الأزرار

**التشخيص:** معظم `TouchableOpacity` (إغلاق، إعدادات، حذف، مشاركة، رجوع، أزرار التقييم) بلا `accessibilityRole`/`accessibilityLabel`؛ قارئ الشاشة يقرأها "زر" بلا معنى. القدوة الجيدة موجودة في `LanguageSwitcher`.

**الإصلاح:** أضف لكل زر:
```tsx
<TouchableOpacity
  accessibilityRole="button"
  accessibilityLabel={t("a11y.deleteReport")}
  onPress={handleDelete}
> ... </TouchableOpacity>
```
وأضف مفاتيح `a11y.*` في ملفات الترجمة. ركّز على: رجوع، إغلاق، إعدادات، حذف، مشاركة، PDF، بدء المراقبة، تبديل اللغة، أزرار التقييم.

**معيار النجاح:** تشغيل VoiceOver/TalkBack ينطق وصفاً مفهوماً لكل زر.

### 🟡 U-7 — زر "حذف الكل" خطر

**التشخيص:** في رأس الرئيسية، زر حذف كل التقارير (🗑) بجانب الإعدادات مباشرةً — سهل الضغط عليه بالخطأ، وإجراء مدمّر. يوجد تأكيد (Alert) لكن مكانه بارز جداً.

**الإصلاح:** انقله إلى داخل شاشة الإعدادات تحت قسم "منطقة الخطر" (Danger Zone)، أو اجعله أقل بروزاً (أيقونة باهتة + يتطلب ضغطة مطوّلة). أبقِ تأكيد الحذف.

### 🟡 U-8 — لا يوجد شريط تنقّل سفلي

**التشخيص:** `app/(tabs)/_layout.tsx` مجرّد `<Slot/>` بلا أي Tab Bar، رغم أن اسم المجلد `(tabs)`. التطبيق فعلياً تنقّل Stack (رئيسية ← جلسة/إعدادات/تقرير). للتطبيق الحالي (3–4 شاشات) هذا مقبول، لكنه يقلّل اكتشاف الميزات.

**الإصلاح (اختياري):** إمّا:
- إبقاء البنية الحالية وإزالة التسمية المضلّلة `(tabs)`، أو
- إضافة Tab Bar حقيقي (`expo-router` Tabs) بثلاثة أقسام: الرئيسية / السجل / الإعدادات — بنفس الألوان (`colors.primary` للنشط، `colors.mutedForeground` للخامل) ومع `flipIconName` للأسهم.

> قرار منتج. إن لم يُطلب، أبقِ Stack لكن وثّق ذلك.

### 🟡 U-9 — تدقيق الأسهم والتنقّل (RTL)

**الوضع الحالي جيد عموماً:**
- أزرار الرجوع تستخدم `flipIconName("arrow-left", isRTL)` → تنقلب صحيحاً في العربية. ✅
- `ReportCard` يستخدم `isRTL ? "chevron-left" : "chevron-right"` (الاتجاه الأمامي صحيح في RTL). ✅
- أسهم "ابدأ" في الرئيسية تستخدم `flipIconName`. ✅

**ما يجب التحقق منه يدوياً:**
1. كل شاشة فيها زر رجوع: تأكد أن السهم يشير "للخارج" الصحيح في كلا اللغتين.
2. أي `chevron`/`arrow` جديد يجب أن يمرّ عبر `flipIconName` أو شرط `isRTL`.
3. ترتيب عناصر الهيدر (`flexDirection: rtl.flexDirection`) — تأكد أن زر الرجوع يبقى في الجهة المتوقّعة (يمين في العربي).

**معيار النجاح:** التنقّل في العربية والإنجليزية يبدو طبيعياً، والأسهم تشير للاتجاه الصحيح دائماً.

### 🟡 U-10 — تقسيم شاشة التقرير الطويلة

**التشخيص:** `report/[id].tsx` تمرير واحد طويل (ملخّص، رسم، سرعة، مسؤولية، سيناريو، عوامل، تقييم، بلاغ خاطئ، ملاحظة قانونية). يصعب التنقّل بين الأقسام.

**الإصلاح:** أضف عناوين أقسام واضحة (موجودة جزئياً) + اختياري: شريط روابط سريع أعلى الشاشة (Summary chips تقفز للقسم)، أو اطوِ الأقسام الثانوية (تحليل الطرف الآخر/العوامل) في Accordion. حافظ على الألوان والمسافات الحالية.

## 4. تحسينات تجربة المستخدم (UX) — مقترحات بنفس الهوية

1. **لمسات Haptics متّسقة:** موجودة في أماكن (بدء المراقبة، التقييم). وحّدها: كل إجراء رئيسي يعطي اهتزازاً خفيفاً، والحادث يعطي اهتزاز تنبيه (موجود).
2. **حالة "جاري المعايرة":** بعد إصلاح المحرك (A-1)، الكشف معطّل أول ~5 ثوانٍ. أضف مؤشّراً صغيراً في شاشة الجلسة "جاري معايرة الحساسات…" حتى يفهم المستخدم سبب عدم البدء الفوري. (يربط UI بمنطق `isEngineReady`).
3. **شفافية الثقة (يربط A-6):** في شاشة التقرير، إذا كانت النتيجة غير قاطعة (`liabilityConclusive=false`) أو الاتجاه غير معاير (`directionCalibrated=false`)، أظهر شارة "نتيجة تقديرية — تحتاج مراجعة" بدل رقم جريء. استخدم `colors.warning`.
4. **تأكيد الإجراءات المدمّرة:** الحذف وبلاغ "ليس حادثاً" لهما تأكيد — جيد. حافظ عليه.
5. **رسائل فارغة أوضح:** الحالة الفارغة جيدة؛ أضف رسماً/أيقونة أكبر ودعوة واضحة (موجودة جزئياً).
6. **أحجام اللمس:** زر الإغلاق 38px — ارفعه إلى 44px (الحد الأدنى الموصى به للمس).
7. **تباين الأزرار النصية الملوّنة:** نصوص مثل `#34C759` على خلفية `#34C75922` تباينها منخفض — استخدم درجة النص الأغمق (`colors.success` على خلفية فاتحة جداً) وتحقّق ≥ 4.5:1.

## 5. تحسينات التعريب (عربي/إنجليزي)

1. **اكتمال الترجمة:** بعد U-3، شغّل فحصاً يقارن مفاتيح `ar.json` و`en.json` (يجب أن تتطابق المفاتيح):
```bash
node -e "const a=require('./lib/locales/ar.json'),e=require('./lib/locales/en.json');const f=(o,p='')=>Object.entries(o).flatMap(([k,v])=>typeof v==='object'&&v?f(v,p+k+'.'):[p+k]);const A=new Set(f(a)),E=new Set(f(e));console.log('في ar وليست en:',[...A].filter(x=>!E.has(x)));console.log('في en وليست ar:',[...E].filter(x=>!A.has(x)));"
```
   يجب أن تكون القائمتان فارغتين.
2. **لا جُمل مركّبة في الكود:** استخدم interpolation (`t("key", { value })`) بدل لصق سلاسل، لأن ترتيب الكلمات يختلف بين اللغتين.
3. **الأرقام والتواريخ:** مرّر كل رقم معروض عبر `formatNumber/formatDecimal/formatPercentage/formatDate` (U-4).
4. **اتجاه الأيقونات النصية:** الأسهم عبر `flipIconName` (U-9). الأيقونات غير الاتجاهية (zap, shield, alert) لا تُقلب — صحيح.
5. **اختبار لغوي:** بدّل اللغة من الإعدادات وتصفّح: الرئيسية، الجلسة، التقرير، الإعدادات، نافذة البلاغ الخاطئ، الإشعار. تأكد لا اختلاط لغتين في شاشة واحدة.
